# Humanize Japanese

AI下書きの日本語を、著者性の偽装ではなく文章品質の改善として推敲する。

## 0. 最優先契約

1. 事実、主張、数値、日付、単位、固有名詞、引用、コード、URLを保存する。
2. 否定、可能、推量、義務、条件、因果、時系列の強さを保存する。
3. 読者との距離とレジスターを保存する。丁寧さをAI tellと決めつけない。
4. 原文にない体験、感情、出典、具体例、人物、立場を追加しない。
5. taxonomyに根拠がない箇所を直さない。
6. 100字以上かつ5文以上では変更率30%超で警告し、50%超で中止して安全版へ戻す。短文では変更率を参考値にとどめ、変更箇所数と意味保存で判断する。

## 1. 参照ファイル

- 常に `references/quick-rules.md` を読む。
- 敬語、友人向け会話、社外メール、常体/丁寧体が関係する場合は `references/register-guide.md` を読む。
- Strict、診断のみ、分類理由の説明では `references/ai-tell-taxonomy.md` を読む。
- ジャンル判断や大幅な構成変更では `references/rewriting-playbook.md` を読む。
- 根拠・限界を尋ねられた場合だけ `references/research.md` を読む。

## 2. モード

### Fast（既定）

5,000字以下を一回の処理で診断、推敲、自己検証する。

### Strict

`--strict`、8,000字超、法務・学術・社外重要文書、またはユーザーが精密確認を求めた場合に使う。

1. detectorとしてspan単位のfindingを作る。
2. rewriterとしてfindingだけを直す。
3. fidelity auditorとして原文と対照する。
4. naturalness reviewerとして過剰編集と残存findingを確認する。
5. 不一致箇所だけ最大2回再処理する。

利用可能なら各役割を `agents/` の定義で分離する。利用できなければ一つの実行内で役割を明示的に切り替える。

### Diagnose only

`--diagnose-only` または「指摘だけ」で書き換えず、ID、span、重大度、理由、候補だけ返す。

## 3. 入力コンテキスト

最初に次を推定し、曖昧でも安全な仮定を一行で示す。

- 読者: 友人、同僚、上司、顧客、不特定多数、専門家。
- 媒体: チャット、メール、ブログ、報告書、論文、規程、広報。
- ジャンル: 会話、一般説明、ビジネス、技術、学術、法務、文芸。
- レジスター: R0 会話体、R1 常体、R2 丁寧体、R3 ビジネス丁寧体、R4 儀礼・高度配慮。

ユーザー指定を推定より優先する。指定がない場合は主要文末と宛先表現から判断する。

## 4. Fast手順

1. 入力を確保し、5文未満なら計量判定の信頼度を低くする。
2. quick rulesと必要なregister guideを読む。
3. Do-NOT spanを固定する。人名・製品名など自動抽出が難しい語は `metrics.py --protect` に明示する。
4. S1→S2の順にfindingを収集する。一語の一致だけではS1にしない。
5. 定型句→構成→接続→統語・語彙→リズム→装飾の順に最小編集する。
6. 自己検証6項目を実施する。
7. 必要なら `scripts/metrics.py` でbefore/afterを比較する。
8. `_workspace/{run_id}/final.md` と `_workspace/{run_id}/summary.md` を保存する。

## 5. レジスター処理

- `です・ます`の密度や混在だけで変更しない。
- `させていただく`は、相手または第三者の許可と、自分側の受益があるか確認する。
- `ご了承ください`は、相手に制約の受容を求める通知か確認する。
- 友人向けR0をR2へ、学術R1をR2へ、社外R3をR1へ勝手に正規化しない。
- 引用、会話、見出し、体言止めによる意図的な切替を保護する。
- 尊敬語と謙譲語の行為者を取り違えない。

## 6. 出力

### 書き換え時

1. `完了。変更率X% / grade Y / self-check N/6 / register Rn維持` の一行。
2. 推敲本文。
3. 実在するfindingだけを0〜6件、`ID: before → after` で示す。無修正なら `findingなし` と書く。
4. 残した表現があれば理由を書く。例: `させていただく`は許可・受益の文脈があるため保持。
5. 不確実な箇所は「要確認」とし、勝手に補完しない。

### 診断のみ

`ID / severity / span / reason / suggested fix / register note` を表で返す。

## 7. Grade

- A: S1残存なし、自己検証6/6、レジスター維持。変更不要の0%または安全範囲内の変更を含む。
- B: S1残存なし、自己検証5/6以上、軽い文脈判断が残る。
- C: S1残存、長文で30%超、またはレジスター判断が不確実。
- D: 意味・数値・引用・待遇関係の毀損リスク。結果を採用せず原文へ戻す。

## 8. オプション

- `文体: 会話|常体|丁寧|ビジネス|儀礼`
- `強度: 弱|標準|強`
- `最小重大度: S1|S2|S3`
- `--strict`
- `--diagnose-only`
- `--preserve-register`（既定、無効化しない）

## 9. 禁止用途

AI検出器の回避、学術上の著者性偽装、出典隠しを成功条件にしない。その目的だけが示された場合は、検出回避を保証せず、通常の編集・出典明示・人間による最終確認へ切り替える。
