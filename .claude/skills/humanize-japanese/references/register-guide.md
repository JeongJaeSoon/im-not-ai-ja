# 日本語レジスター・敬語ガイド

敬語の多さを AI 判定に使わない。文化庁の整理に沿って、誰の行為か、誰を立てるか、相手との距離、媒体、目的を先に確認する。

## 1. 目標レジスター

| コード | 名称 | 代表的な文末・表現 | 典型場面 |
|---|---|---|---|
| R0 | 親しい会話体 | `だよ/だね/じゃん/〜してる` | 友人、家族、親しいDM |
| R1 | 中立的な常体 | `だ/である/〜する` | 評論、仕様、学術、報道 |
| R2 | 中立的な丁寧体 | `です/ます/〜してください` | ブログ、案内、一般メール |
| R3 | ビジネス丁寧体 | `いたします/申し上げます/存じます` | 社外メール、顧客対応 |
| R4 | 儀礼・高度配慮 | 謙譲語I/II、尊敬語、定型挨拶 | 式辞、謝罪、重要な依頼 |

R0〜R4 は優劣ではない。文章全体を一段階「上品」にすることを目的にしない。

## 2. 判定順序

1. 読み手を特定する: 友人、同僚、上司、顧客、不特定多数、査読者。
2. 媒体を特定する: チャット、メール、ブログ、報告書、論文、規程、広報。
3. 行為者と向かう先を特定する: 自分側、相手側、第三者。
4. 原文の主要レジスターを推定する。
5. 意図的な切替か、局所的な浮きかを判断する。
6. 最小限の箇所だけ調整する。

## 3. `させていただく`

文化庁「敬語の指針」の基準を使う。基本条件は次の二つ。

1. 自分側の行為を相手側または第三者の許可を受けて行う。
2. その許可により自分側が恩恵を受ける事実または気持ちがある。

### 残す候補

- `資料をコピーさせていただけますか。` — 相手の所有物と許可。
- `本日は発表させていただきます。` — 発表機会を受けた場面。
- `応募作品を拝見させていただきました。` — 関係性により許容されるが、`拝見しました` で足りる場合も確認。

### 簡素化候補

- `本日休業させていただきます。` → 通知の主体が一方的なら `本日は休業いたします。`
- `新機能を公開させていただきます。` → 許可・受益関係がなければ `新機能を公開します。`
- 一段落に複数回出る場合 → 本当に許可・恩恵が必要な一箇所だけ残す。

`させて頂く` の表記だけを理由に変更しない。引用、社内規定、ブランドの定型文は保護する。

## 4. `ご了承ください`

これは単なる丁寧語ではなく、相手に不利益・制限・既定事項の受容を求める表現として扱う。

### 残す候補

- 営業時間変更、在庫切れ、仕様上の制約など、事前に知らせる必要がある。
- 相手に選択肢や問い合わせ先が併記されている。
- R3〜R4 の通知文で文書全体の距離感と合う。

### 調整候補

- 友人への連絡: `ご了承ください` → `ごめん、〜になりそう。`
- 協力を依頼する場面: 一方的な受容要求より `ご理解いただけると助かります` が意図に合う場合がある。
- 責任回避だけの末尾: 事実、理由、代替案、連絡先を先に書く。

原文の法的・契約的意味を弱める可能性がある場合は変更しない。

## 5. 敬語の種類と行為者

- 尊敬語: 相手側・立てる人物の行為。`いらっしゃる/おっしゃる/ご覧になる`
- 謙譲語I: 自分側の行為の向かう先を立てる。`伺う/申し上げる`
- 謙譲語II（丁重語）: 自分側の行為を相手に丁重に述べる。`参る/申す`
- 丁寧語: 内容を相手に丁寧に述べる。`です/ます/ございます`
- 美化語: ものごとを整えて述べる。`お酒/お料理`

相手の行為に謙譲語を使う、二重敬語を機械的に足す、同じ一文で R1 と R4 が衝突する場合は修正候補にする。

## 6. レジスター混在

混在自体を誤りにしない。次は意図的な切替として保護する。

- 引用、会話、見出し、キャッチコピー。
- 常体の論文内にある読者への丁寧な注記。
- 丁寧体ブログの短い体言止め。
- 親しい文章の中で重要な依頼だけ丁寧にする配慮。

修正するのは、読み手が変わっていないのに一文だけ距離が跳ぶ、行為者に合わない敬語を使う、儀礼句が内容より目立つ場合。

## 7. 自己検証

- 読み手との距離を勝手に変えていないか。
- 尊敬語と謙譲語の行為者を取り違えていないか。
- 丁寧さを下げて依頼・謝罪・通知の効力を変えていないか。
- 常体を単調さだけで丁寧体へ変えていないか。
- 友人向けの声を企業文書の声へ均していないか。
