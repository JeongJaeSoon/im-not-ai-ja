# Humanize Japanese quick rules

Fast モードではこのファイルだけを読み、必要な箇所だけ直す。

## 絶対条件

1. 数値、日付、単位、固有名詞、引用、コード、URLを変えない。
2. 否定、可能、推量、義務、条件、因果の強さを変えない。
3. 原文にない体験、感情、出典、具体例を足さない。
4. レジスターを先に判定し、丁寧さを AI tell として削らない。
5. 変更率30%超の警告と50%超の中止は100字以上かつ5文以上に適用する。短文では参考値として扱う。

## 優先順

1. S1 の定型句・誇張: `これにより`、`重要です`、`期待されます`、`浮き彫り`、`新たな可能性`。
2. 構成の型: 無理な三点、予告と再要約、`課題と展望`。
3. 接続と照応: 文頭接続詞、`これは/そのため` の反復。
4. 統語と語彙: 主語過剰、false agency、漢語連鎖、飾りのカタカナ。
5. リズム: 最低5文ある場合だけ文長 CV、読点、文末反復を確認。
6. 書式: 太字、絵文字、コロン、ダッシュ、不要な括弧。

## レジスター

- R0 親しい会話体、R1 常体、R2 丁寧体、R3 ビジネス丁寧体、R4 儀礼・高度配慮。
- `させていただく`: 許可と受益があるか。なければ `します/いたします` を検討。
- `ご了承ください`: 制約の受容を求める通知か。友人向けや協力依頼なら距離感を再検討。
- 敬語の行為者: 相手の行為に謙譲語を使わず、二重敬語を機械的に足さない。
- `です・ます` の密度や混在だけで修正しない。

## 6項目の自己検証

- [ ] 事実・数値・固有名詞・引用が保存されている。
- [ ] モダリティと因果が保存されている。
- [ ] 目標レジスターと行為者が保存されている。
- [ ] 原文にない「人間らしさ」を追加していない。
- [ ] S1 が残っていない、または残す理由がある。
- [ ] 変更率が安全範囲にある。
